# Product Requirements Document (PRD)

## Version

0.1

## Projet

Market Intelligence Platform

## Date

23 juillet 2026

---

# 1. Introduction

Market Intelligence Platform est une plateforme d'aide à la décision destinée aux vendeurs e-commerce.

L'objectif est de permettre aux utilisateurs d'analyser les opportunités présentes sur différentes marketplaces afin de prendre des décisions basées sur les données avant d'investir dans un produit.

Le produit vise à transformer des données de marché complexes en informations simples et exploitables.

---

# 2. Objectif du produit

Market Intelligence Platform a pour objectif de réduire l'incertitude liée au choix d'un produit à commercialiser.

La plateforme doit permettre aux utilisateurs de :

- analyser un marché ;
- identifier des tendances ;
- comparer plusieurs produits ;
- évaluer le potentiel commercial ;
- prendre une décision avant un investissement.

---

# 3. Le problème

Les nouveaux vendeurs e-commerce disposent aujourd'hui d'un grand nombre d'informations, mais il est difficile d'évaluer objectivement une opportunité.

Les décisions sont souvent prises à partir :

- d'intuitions ;
- de tendances observées sur les réseaux sociaux ;
- de recommandations non vérifiées.

Cette approche peut conduire à investir dans des produits peu rentables ou fortement concurrentiels.

Market Intelligence Platform répond à ce problème en proposant une analyse structurée basée sur les données.

---

# 4. Utilisateurs cibles

## Utilisateur principal

La première utilisatrice est la fondatrice du projet.

La plateforme sera utilisée dans un premier temps pour analyser des opportunités réelles de vente en ligne et tester la pertinence des analyses produites.

## Utilisateurs cibles à moyen terme

- vendeurs Amazon ;
- vendeurs Etsy ;
- vendeurs Cdiscount ;
- vendeurs Fnac ;
- entrepreneurs e-commerce.

## Utilisateurs potentiels à long terme

- petites entreprises ;
- commerçants ;
- entrepreneurs internationaux ;
- vendeurs souhaitant développer une activité en ligne, notamment en Afrique.

---

# 5. Cas d'utilisation

## Cas d'utilisation 1 : Analyser un produit

### Situation

Un utilisateur souhaite savoir si un produit représente une opportunité commerciale.

### Action

L'utilisateur recherche un produit dans la plateforme.

### Résultat attendu

La plateforme affiche :

- prix moyen ;
- évolution du prix ;
- nombre de vendeurs ;
- niveau de concurrence ;
- tendances ;
- potentiel de marge ;
- score d'opportunité.

---

## Cas d'utilisation 2 : Comparer plusieurs produits

### Situation

Un utilisateur hésite entre plusieurs produits.

### Action

Il compare plusieurs opportunités.

### Résultat attendu

La plateforme classe les produits selon différents indicateurs afin de faciliter la décision.

---

## Cas d'utilisation 3 : Suivre l'évolution d'un produit

### Situation

Un vendeur souhaite suivre un marché dans le temps.

### Résultat attendu

La plateforme permet d'observer :

- évolution des prix ;
- apparition de nouveaux vendeurs ;
- évolution de la tendance.

---

# 6. Fonctionnalités principales

La plateforme devra progressivement permettre :

- rechercher un produit ;
- collecter les données associées ;
- comparer plusieurs produits ;
- visualiser les tendances ;
- analyser la concurrence ;
- estimer le potentiel commercial ;
- afficher un tableau de bord décisionnel.

---

# 7. Données et indicateurs

Pour chaque produit analysé, la plateforme pourra afficher :

- prix actuel ;
- évolution historique du prix ;
- nombre de vendeurs ;
- niveau de concurrence ;
- tendance du marché ;
- popularité ;
- estimation de marge ;
- score global d'opportunité.

---

# 8. Market Opportunity Score

Le Market Opportunity Score est un indicateur synthétique permettant d'évaluer le potentiel commercial d'un produit.

Son objectif est de transformer plusieurs informations du marché en une analyse simple permettant de comparer différentes opportunités.

Le score pourra prendre en compte plusieurs critères :

- tendance du produit ;
- évolution du prix ;
- concurrence ;
- nombre de vendeurs ;
- potentiel de marge ;
- stabilité du marché.

Les premières versions utiliseront une approche basée sur des règles définies.

Les versions futures pourront intégrer des méthodes statistiques ou des modèles de Machine Learning.

---

# 9. MVP (Minimum Viable Product)

La première version utilisable devra permettre :

- sélectionner un produit ;
- récupérer des données associées ;
- stocker les informations ;
- calculer des indicateurs simples ;
- afficher un premier tableau de bord ;
- comparer plusieurs opportunités.

L'objectif du MVP est de valider la capacité de la plateforme à aider dans la prise de décision.

---

# 10. Hors périmètre initial

Les fonctionnalités suivantes ne seront pas prioritaires dans les premières versions :

- automatisation complète de la vente ;
- gestion des commandes ;
- gestion des stocks ;
- marketplace interne ;
- fonctionnalités avancées d'intelligence artificielle.

Ces fonctionnalités pourront être étudiées dans des versions futures.

---

# 11. Critères de réussite

Le projet sera considéré comme réussi lorsque :

- la plateforme permettra d'analyser un produit de manière structurée ;
- les données seront collectées et transformées automatiquement ;
- un utilisateur pourra comparer plusieurs opportunités ;
- les indicateurs permettront de faciliter une décision d'investissement ;
- l'architecture pourra évoluer vers une version plus complète.
